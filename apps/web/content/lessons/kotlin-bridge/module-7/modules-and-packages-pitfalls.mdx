**`public`** is the default in Kotlin (watch Java habits — package privacy does not exist the same way).

**`internal`** shrinks API surface to a **module**.

**`private`** at file top-level hides helpers inside that file.

When mixing Java and Kotlin in one Gradle module, **`internal`** still protects against Java in **other** modules — within same module, Java can access Kotlin `internal` via synthetic accessors (compiler-dependent rules) — treat **`internal`** as team-wide for a module.

## Recap

- **`internal`** expresses artifact boundaries — not Java package boundaries.
- Plan library APIs deliberately — Java consumers see **`public`** Kotlin members most clearly.
