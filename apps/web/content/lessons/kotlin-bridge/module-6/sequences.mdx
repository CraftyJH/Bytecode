**`Sequence<T>`** is lazy. Intermediate ops build a recipe; **terminal** ops run it.

```kotlin
val firstBig = items.asSequence()
    .map { expensive(it) }
    .first { it.score > 0.9 }
```

## Recap

- Use **`asSequence()`** when chaining many passes over large data or when **`take`** early-stops work.
- Remember **single-pass** behavior — cannot iterate the same sequence twice without rebuilding.
