Top-level **`fun`** replaces many `public static` utility methods in Java. Place helpers in a file without a wrapper class — the compiler still maps names for the JVM.

## Default and named arguments

```kotlin
fun connect(host: String = "localhost", port: Int = 8080) { ... }

connect()                       // both defaults
connect(port = 9000)            // host default, name the port
connect(host = "db.internal")   // partial override with clarity at call site
```

## Single-expression functions

```kotlin
fun double(x: Int): Int = x * 2
// or
fun double(x: Int) = x * 2
```

## `Unit`

If you omit a return type and use block body without `return`, Kotlin infers **`Unit`**.

## Why this matters in real code

Function signatures are where API quality starts. Java developers often add overloads for convenience:

- `fetch()`
- `fetch(timeout)`
- `fetch(timeout, retries)`

Kotlin defaults collapse these into one declaration while keeping call sites explicit with named arguments. This tends to reduce accidental parameter swaps (`host, port` style bugs) and makes code reviews easier because intent is visible at the call site.

```kotlin
fun fetch(host: String = "localhost", port: Int = 8080, retries: Int = 2) { ... }

fetch(port = 9090)          // clear intent
fetch(retries = 0)          // no hidden meaning
```

## Java interop caution

Defaults are Kotlin language features, not JVM bytecode features. Java callers do not see default arguments automatically. If a function is called from Java, plan for either:

- explicit overloads, or
- `@JvmOverloads` in the interop module.

Treat this as contract design: Kotlin ergonomic API for Kotlin callers, stable overload surface for Java callers.

## Common mistakes

- Expecting Java callers to see default parameters without **`@JvmOverloads`** or explicit overloads.
- Returning **`Unit`** accidentally from branchy lambdas when you meant a value — watch last-expression rules.

## Recap

- **Defaults + named args** cut overload explosion compared to Java.
- **`Unit`** is Kotlin’s “no value” for typing purposes.
