Extensions add **namespaced functions** without subclassing:

```kotlin
fun String.isDigitOnly(): Boolean = all { it.isDigit() }
```

Inside, **`this`** refers to the receiver.

## Scope and imports

Extensions live in packages — import them like top-level functions:

```kotlin
import com.example.utils.wordCount
```

## Extensions vs inheritance

They **do not override** Java methods on the receiver’s class hierarchy. Accidentally creating two extensions with the same name on `Base` vs `Derived` trips people migrating from Java polymorphism.

## Recap

- Syntax sugar with **receiver type** as first parameter under the hood.
- **Dispatch rules differ** from Java `virtual` methods — design APIs deliberately.
