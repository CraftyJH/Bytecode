Lambdas in Kotlin look like Java’s, with conveniences:

```kotlin
list.filter { it % 2 == 0 }
```

**Trailing lambda** syntax moves the lambda outside parentheses when it is the last argument:

```kotlin
thread {
    println("running")
}
```

## Method references

Java:

```java
names.sort(String::compareToIgnoreCase);
```

Kotlin often uses lambdas or references similarly:

```kotlin
names.sortBy { it.lowercase() }
```

## SAM recap

When a Java API expects a **single abstract method** interface, Kotlin lets you pass a lambda where the compiler adapts it — just like `Runnable` in Java 8.

## Common mistakes

- Shadowing **`it`** in nested lambdas — name outer parameters explicitly.
- Passing **`null`** as a functional value without a nullable functional type — signature matters.

## Recap

- **`it`** is shorthand for unary lambdas.
- **SAM conversion** connects Kotlin lambdas to Java functional interfaces.
