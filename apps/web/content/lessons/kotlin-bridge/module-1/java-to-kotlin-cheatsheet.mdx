This lesson is a **fast orientation map** — not exhaustive. Keep it nearby while you work through later modules. Prefer **expression-oriented** Kotlin where it clarifies data flow; prefer **statements** where side effects and logging dominate.

## Variables

| Java | Kotlin |
|------|--------|
| `final int x = 1;` | `val x = 1` |
| `int y = 2; y = 3;` | `var y = 2; y = 3` |

Type annotations are optional when inference is clear: `val name: String = "Ada"`.

## Strings

| Java | Kotlin |
|------|--------|
| `"Hello, " + name` | `"Hello, $name"` or `"Hello, ${name.uppercase()}"` |
| `String.format("%02d", n)` | string templates or `n.toString().padStart(2, '0')` |

## Functions

| Java | Kotlin |
|------|--------|
| `public static int add(int a, int b)` | `fun add(a: Int, b: Int): Int` (top-level) |
| overloads for defaults | default parameters (`fun greet(name: String = "World")`) |

## Classes (preview)

| Java | Kotlin |
|------|--------|
| verbose POJO + getters | `data class User(val name: String)` |
| `enum` with behavior | `enum class` with methods |

## Nullability (preview)

| Java | Kotlin |
|------|--------|
| `@Nullable String` | `String?` |
| defensive `if (s != null)` | `s?.length`, smart casts after checks |

## Exercise strategy

Use the playground to practice **printing** results — mirroring `System.out.println` habits but with **`println`**. Pay attention to **exact output** when the grader compares strings.

## Recap

- **`val` / `var`** replace many `final`/mutable Java locals.
- **String templates** replace noisy concatenation.
- **Expression `if` / `when`** pair well with single-expression functions.
