Java methods without **JSR-305**/**JSpecify**-style nullability show up as **platform types** — flexible at compile time, dangerous if abused.

Strategies:

- Add **nullability annotations** to Java sources or package defaults.
- Wrap returns at the boundary with **`requireNotNull`** or explicit checks.
- Prefer **Kotlin libraries** with precise types for new code.

## Recap

- Platform types relax checking — **you** supply discipline at boundaries.
- **Gradual migration**: tighten types module by module.
