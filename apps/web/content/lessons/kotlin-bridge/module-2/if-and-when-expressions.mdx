Java’s ternary operator has one Kotlin counterpart in **`if` expressions**:

```kotlin
val label = if (score >= 90) "A" else "B"
```

Multi-branch logic uses **`when`**. Unlike Java `switch`, `when` can be **expression-valued** and supports guards:

```kotlin
val grade = when {
    score >= 90 -> "A"
    score >= 80 -> "B"
    else -> "C"
}
```

## Subject vs subject-less `when`

```kotlin
when (x) {
    0 -> println("zero")
    in 1..9 -> println("digit")
    else -> println("other")
}
```

vs.

```kotlin
when {
    x == 0 -> ...
    x in 1..9 -> ...
    else -> ...
}
```

## Common mistakes

- Mixing **statement** `if` (with side effects only) and **expression** `if` — compiler errors when branch types mismatch.
- Forgetting **`else`** in expression `when` when the compiler cannot prove exhaustiveness.

## Recap

- **`if` as expression** replaces most Java ternaries cleanly with readable branching.
- **`when`** generalizes `switch` with richer matching — sealed hierarchies later unlock compiler-checked completeness.
