Ranges are lightweight **`IntRange`** / **`CharRange`** values — not Java streams unless you convert.

## Core operators

- **`a..b`**: inclusive both ends.
- **`a until b`**: includes `a`, excludes `b`.
- **`step`**: adjust stride (`1..10 step 2`).

## Iteration

```kotlin
for (i in 1..5) {
    println(i)
}
```

Indices over arrays/lists:

```kotlin
for (i in arr.indices) {
    println("${i}: ${arr[i]}")
}
```

Or values with index via `withIndex()`:

```kotlin
for ((idx, value) in arr.withIndex()) {
    println("$idx -> $value")
}
```

## Common mistakes

- Using **`until`** when you meant inclusive upper bound — off-by-one bugs migrating from Java `for (i=0;i<n;i++)`.
- Confusing **`downTo`** / **`reversed()`** — descending loops need explicit intent.

## Recap

- **`..` vs `until`** control inclusivity.
- **`for (x in …)`** replaces many indexed Java loops when you only need elements.
