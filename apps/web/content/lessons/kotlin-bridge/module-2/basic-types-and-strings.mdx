Kotlin maps Java primitives to types such as `Int`, `Long`, `Double`, `Boolean`, and `Char`. Under the hood they compile to JVM primitives when possible (`Int` → `int`) but behave like types with methods (`10.inc()`).

## String templates

Instead of:

```java
System.out.println("Hello, " + name + ", count=" + count);
```

Kotlin allows:

```kotlin
println("Hello, $name, count=$count")
```

Expressions need braces:

```kotlin
println("square=${n * n}")
```

## Raw / multiline strings

Triple quotes preserve newlines — handy for SQL snippets or JSON embedded in tests:

```kotlin
val sql = """
    SELECT id, title
    FROM posts
    WHERE owner_id = ?
""".trimIndent()
```

## Common mistakes

- Forgetting braces around **`${expr}`** when `expr` contains operators — Kotlin may parse `${` incorrectly without closing braces.
- Mixing Java `char` thinking with Kotlin **`Char`** vs **`String`** — `'a'` is `Char`, `"a"` is `String`.

## Recap

- Numeric types are explicit (`Int`, `Double`, …) with inference at locals.
- **Templates** replace most `String.format` noise for simple joins.
- **Raw strings** reduce escaping for literals with quotes or windows paths.
