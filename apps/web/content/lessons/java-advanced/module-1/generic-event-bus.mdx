This capstone lesson builds a **type-safe event bus** — a publish/subscribe system where publishers emit events and subscribers receive only the event types they registered for, all without a single unchecked cast in application code. Event buses appear in Android (EventBus library), Spring (ApplicationEventPublisher), Guava (EventBus), and custom microservice frameworks. Building one from scratch shows exactly how every concept from this module — generic classes, type tokens, wildcards, `Class<T>` keys, and `Consumer<T>` handlers — combines into a coherent, production-quality design.

## What an event bus does

An event bus decouples producers from consumers: the producer emits an event object; the bus routes it to every registered handler for that event type; handlers never know who emitted the event.

```
Publisher              EventBus                  Subscriber
──────────             ─────────                 ──────────────
emit(new UserCreated()) ──▶  on(UserCreated.class, handler) ──▶ handler.accept(event)
emit(new OrderPlaced()) ──▶  no handler registered → ignored
```

## Design: `Map<Class<?>, Consumer<?>>` as the registry

The registry maps each event *class* to its handler. `Class<?>` is the key (the type token), `Consumer<?>` is the value. The generic methods use `T` to tie the key and value together at compile time:

```java
import java.util.*;
import java.util.function.*;

public class EventBus {
    // Keys are Class<?> objects; values are Consumer<?> for the matching type
    private final Map<Class<?>, Consumer<?>> handlers = new HashMap<>();

    /**
     * Register a handler for events of type T.
     * A second registration for the same type replaces the first.
     */
    @SuppressWarnings("unchecked")
    public <T> void on(Class<T> eventType, Consumer<T> handler) {
        handlers.put(eventType, handler);
    }

    /**
     * Emit an event. The handler registered for event.getClass() is invoked,
     * if one exists. Unregistered event types are silently dropped.
     */
    @SuppressWarnings("unchecked")
    public <T> void emit(T event) {
        Consumer<T> handler = (Consumer<T>) handlers.get(event.getClass());
        if (handler != null) {
            handler.accept(event);
        }
    }
}
```

The `@SuppressWarnings("unchecked")` on `emit` is safe because `on` enforces the invariant: the handler stored under `Class<T>` is always a `Consumer<T>`. The `(Consumer<T>)` cast cannot fail as long as no one bypasses `on`.

<Note>The single-handler-per-type design is intentional for simplicity. The multi-subscriber variant (next section) stores a `List<Consumer<?>>` per type — the bus delivers to all registered handlers.</Note>

## Supporting multiple subscribers per event type

Replace the single `Consumer<?>` with a `List<Consumer<?>>`:

```java
import java.util.*;
import java.util.function.*;

public class EventBus {
    private final Map<Class<?>, List<Consumer<?>>> handlers = new HashMap<>();

    @SuppressWarnings("unchecked")
    public <T> void on(Class<T> eventType, Consumer<T> handler) {
        handlers.computeIfAbsent(eventType, k -> new ArrayList<>())
                .add(handler);
    }

    @SuppressWarnings("unchecked")
    public <T> void emit(T event) {
        List<Consumer<?>> list = handlers.get(event.getClass());
        if (list != null) {
            for (Consumer<?> h : list) {
                ((Consumer<T>) h).accept(event);
            }
        }
    }

    /** Remove all handlers for an event type. */
    public void off(Class<?> eventType) {
        handlers.remove(eventType);
    }
}
```

`computeIfAbsent` creates the list on first registration and returns the (possibly new) list for immediate `.add(handler)`. This is the `HashMap` grouping pattern from lesson 6.

## Defining event types

Keep event types as immutable records — lightweight, clearly named, carry only what subscribers need:

```java
record UserCreated(String id, String email) {}
record UserDeleted(String id) {}
record OrderPlaced(String orderId, double amount) {}
record PaymentReceived(String orderId) {}
```

## Complete worked example

```java
import java.util.*;
import java.util.function.*;

public class Main {
    // --- Event types ---
    record UserCreated(String id, String email) {}
    record OrderPlaced(String orderId, double total) {}
    record PaymentReceived(String orderId) {}

    // --- EventBus (multi-subscriber) ---
    static class EventBus {
        private final Map<Class<?>, List<Consumer<?>>> handlers = new HashMap<>();

        @SuppressWarnings("unchecked")
        public <T> void on(Class<T> type, Consumer<T> handler) {
            handlers.computeIfAbsent(type, k -> new ArrayList<>()).add(handler);
        }

        @SuppressWarnings("unchecked")
        public <T> void emit(T event) {
            List<Consumer<?>> list = handlers.get(event.getClass());
            if (list != null) list.forEach(h -> ((Consumer<T>) h).accept(event));
        }
    }

    public static void main(String[] args) {
        EventBus bus = new EventBus();

        // Audit log — listens to everything interesting
        bus.on(UserCreated.class, e ->
            System.out.println("[AUDIT] New user: " + e.email()));
        bus.on(OrderPlaced.class, e ->
            System.out.println("[AUDIT] Order " + e.orderId() + " for $" + e.total()));
        bus.on(PaymentReceived.class, e ->
            System.out.println("[AUDIT] Payment for " + e.orderId()));

        // Email service — only user events
        bus.on(UserCreated.class, e ->
            System.out.println("[EMAIL] Welcome email → " + e.email()));

        // Inventory service — only order events
        bus.on(OrderPlaced.class, e ->
            System.out.println("[INVENTORY] Reserve stock for " + e.orderId()));

        // Trigger the workflow
        bus.emit(new UserCreated("u1", "alice@example.com"));
        bus.emit(new OrderPlaced("o42", 199.99));
        bus.emit(new PaymentReceived("o42"));
    }
}
```

Expected output:
```
[AUDIT] New user: alice@example.com
[EMAIL] Welcome email → alice@example.com
[AUDIT] Order o42 for $199.99
[INVENTORY] Reserve stock for o42
[AUDIT] Payment for o42
```

## Common mistakes

- **Putting a bare Consumer into a List-valued map** — if the map value type is `List<Consumer<?>>`, calling `handlers.put(type, handler)` passes a Consumer where a List is expected, causing a compile error. Always use `computeIfAbsent(...).add(handler)`.
- **Emitting an event whose type is a subclass** — `event.getClass()` returns the *concrete* class. If you register under a parent class (`on(Event.class, h)`) and emit a subclass (`emit(new SpecificEvent())`), the handler won't be found. Design event hierarchies carefully or walk the class hierarchy when looking up handlers.
- **Forgetting `@SuppressWarnings("unchecked")` on `emit`** — the cast from `Consumer<?>` to `Consumer<T>` produces an unchecked warning. The cast is safe by design (enforced by `on`), but without the annotation the warning noise obscures other real warnings.
- **Mutating the handler list while emitting** — if a handler modifies the bus (registers or removes a handler), and you're iterating the list in `emit`, you'll get a `ConcurrentModificationException`. Copy the list before iteration: `new ArrayList<>(list).forEach(...)`.

## Module recap

You've completed Module 1: Generics In Depth. Here's what you can now do:

- **Write generic classes** with one or more type parameters, including `Pair<A, B>` and `Box<T>`.
- **Apply upper bounds** (`<T extends Comparable<T>>`) to call methods on unknown types.
- **Use wildcards** correctly — `? extends T` for read-only covariant views, `? super T` for write-capable contravariant views, and `<?>` unbounded for type-agnostic code.
- **Declare generic methods** with their own `<T>` and infer type arguments from context.
- **Explain type erasure** and its consequences: no `new T[]`, no `instanceof Box<String>`, raw types cause heap pollution.
- **Chain Comparators** with `comparing().thenComparing().reversed()` for multi-key sorting.
- **Build generic data structures** (Stack) backed by `Object[]` with the unchecked-cast pattern.
- **Implement generic algorithms** — binary search, merge sort — with Comparable and Comparator bounds.
- **Use `Class<T>` as a type token** for runtime-safe heterogeneous containers.
- **Compose all of the above** into a type-safe event bus with `Consumer<T>` handlers and `Class<?>` keys.
