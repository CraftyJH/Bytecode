Terminal operations are where stream pipelines actually *do* something. Until a terminal operation runs, no element has been processed — the pipeline is just a recipe. Terminal operations trigger execution, consume the stream, and produce a result (a value, a side effect, or a collection). This lesson covers the full set: counting, matching, finding, reducing, and primitive stream arithmetic.

## forEach and count

The simplest terminals:

```java
List<String> names = List.of("Alice", "Bob", "Carol");

// forEach — side effect, returns void
names.stream().forEach(System.out::println);

// count — number of elements that reach the terminal
long n = names.stream().filter(s -> s.length() > 3).count(); // 2
```

`forEach` processes every element for its side effect. `count` is equivalent to `reduce(0L, (acc, e) -> acc + 1)` but may be optimised by the JVM.

## Primitive stream arithmetic: sum, average, min, max

Generic `Stream<T>` has no arithmetic — boxing `Integer` objects just to add them is wasteful. Use primitive streams:

```java
import java.util.stream.*;

int sum  = IntStream.of(1, 2, 3, 4, 5).sum();         // 15
double avg = IntStream.of(1, 2, 3, 4, 5).average()
               .orElse(0.0);                           // 3.0
int min  = IntStream.of(4, 1, 9, 2).min().orElseThrow(); // 1
int max  = IntStream.of(4, 1, 9, 2).max().orElseThrow(); // 9
```

`min()` and `max()` return `OptionalInt` because the stream may be empty. Always handle the optional.

When starting from a `Stream<T>`, bridge with `mapToInt`:

```java
List<String> words = List.of("hi", "hello", "world");
int totalChars = words.stream().mapToInt(String::length).sum(); // 12
```

## reduce

`reduce` folds the stream to a single value using a binary operation.

**Two-argument form** (always returns the type, uses identity):

```java
int sum     = IntStream.rangeClosed(1, 5).reduce(0, Integer::sum); // 15
int product = IntStream.rangeClosed(1, 5).reduce(1, (a, b) -> a * b); // 120
```

**One-argument form** (returns Optional — handles empty stream gracefully):

```java
Optional<Integer> max = Stream.of(3, 1, 4, 1, 5).reduce(Integer::max);
max.ifPresent(System.out::println); // 5
```

<Note>The identity value must be neutral for the operation: `identity op x == x` for all x. Zero for addition, one for multiplication, empty string for concatenation. Using a wrong identity silently corrupts results on empty streams.</Note>

## findFirst and findAny

Both return an `Optional<T>`. `findFirst` respects encounter order; `findAny` may return any element (faster in parallel streams):

```java
List<Integer> nums = List.of(1, 2, 3, 4, 5);

Optional<Integer> firstEven = nums.stream()
    .filter(n -> n % 2 == 0)
    .findFirst();                          // Optional[2]

System.out.println(firstEven.orElse(-1)); // 2
```

If no element matches the filter, `findFirst()` returns `Optional.empty()`.

## anyMatch, allMatch, noneMatch

These short-circuit terminals stop as soon as the result is determined:

```java
List<Integer> nums = List.of(-2, 0, 3, 5, -1);

boolean anyNeg  = nums.stream().anyMatch(n -> n < 0);  // true  (stops at -2)
boolean allPos  = nums.stream().allMatch(n -> n > 0);  // false (stops at -2)
boolean noneNaN = nums.stream().noneMatch(Double::isNaN); // true
```

`anyMatch` is `true` if at least one element satisfies the predicate — equivalent to `filter(pred).findFirst().isPresent()` but potentially faster.

## Try it

Try these three terminals on the same list:

```java
import java.util.*;
import java.util.stream.*;

public class Main {
    public static void main(String[] args) {
        List<Integer> nums = List.of(4, 7, 2, 9, 1, 6, 3, 8, 5);

        // Sum with reduce
        int total = nums.stream().reduce(0, Integer::sum);
        System.out.println("sum: " + total); // 45

        // Find first even number
        Optional<Integer> firstEven = nums.stream()
            .filter(n -> n % 2 == 0)
            .findFirst();
        System.out.println("first even: " + firstEven.orElse(-1)); // 4

        // Check all positive
        boolean allPositive = nums.stream().allMatch(n -> n > 0);
        System.out.println("all positive: " + allPositive); // true
    }
}
```

After running, change `reduce(0, Integer::sum)` to `mapToInt(Integer::intValue).sum()` — both produce the same result but the second avoids boxing.

## Common mistakes

- **Using the two-argument `reduce` without a valid identity** — `reduce(0, Integer::max)` is wrong for lists with negative numbers; 0 is not neutral for `max`. Use the one-argument form which returns an `Optional`.
- **Calling `findFirst()` without handling the Optional** — `findFirst().get()` throws `NoSuchElementException` on empty results. Always use `orElse`, `orElseGet`, or `orElseThrow`.
- **Using `forEach` when you need a return value** — `forEach` is for side effects and returns `void`. If you need to aggregate, use `reduce`, `collect`, or a specific terminal.
- **Expecting `anyMatch` to always scan the whole stream** — it short-circuits. If you're relying on a side effect inside the predicate for every element, use `forEach` instead.

## Recap

- `forEach(action)` runs a side effect on every element; returns `void`
- `count()` returns the number of elements reaching the terminal
- `sum()`, `average()`, `min()`, `max()` live on primitive streams (`IntStream`, etc.)
- `reduce(identity, op)` folds elements to a single value; use the one-arg form to get `Optional`
- `findFirst()` / `findAny()` return `Optional<T>` — handle the empty case
- `anyMatch` / `allMatch` / `noneMatch` short-circuit for efficiency
