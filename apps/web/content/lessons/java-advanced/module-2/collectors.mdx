The `collect` terminal operation is how you pour a stream into a container — a list, a set, a map, or even a custom data structure. The `Collectors` utility class provides factory methods for the most common aggregation patterns. Knowing this API well transforms you from someone who can only `collect(Collectors.toList())` to someone who can express complex grouping and aggregation in a single readable expression.

## toList and toUnmodifiableList

```java
import java.util.stream.*;

List<String> mutable    = stream.collect(Collectors.toList());
List<String> immutable  = stream.collect(Collectors.toUnmodifiableList());

// Java 16+ shorthand (always unmodifiable):
List<String> immutable2 = stream.toList();
```

Prefer `toUnmodifiableList()` / `toList()` when the caller should not modify the result. Pass `Collectors.toList()` when you need a mutable list (e.g., to pass to a method that calls `add`).

## joining

`joining()` concatenates `Stream<String>` elements:

```java
String csv = Stream.of("Alice", "Bob", "Carol")
    .collect(Collectors.joining(", "));           // "Alice, Bob, Carol"

String html = Stream.of("Alice", "Bob", "Carol")
    .collect(Collectors.joining(", ", "<ul>", "</ul>"));
// "<ul>Alice, Bob, Carol</ul>"
```

The three-argument form adds a prefix and suffix around the entire result — not around each element.

## groupingBy

`groupingBy(classifier)` partitions elements into groups by the value returned by the classifier function. The result is a `Map<K, List<T>>`:

```java
List<String> words = List.of("apple", "banana", "avocado", "blueberry", "apricot");

Map<Character, List<String>> byLetter = words.stream()
    .collect(Collectors.groupingBy(s -> s.charAt(0)));
// {a=[apple, avocado, apricot], b=[banana, blueberry]}
```

A *downstream collector* processes the values within each group before they're placed in the map:

```java
// Count per group
Map<Character, Long> counts = words.stream()
    .collect(Collectors.groupingBy(s -> s.charAt(0), Collectors.counting()));

// Uppercase values in each group
Map<Character, List<String>> upper = words.stream()
    .collect(Collectors.groupingBy(
        s -> s.charAt(0),
        Collectors.mapping(String::toUpperCase, Collectors.toList())
    ));
```

<Note>The key iteration order of the map returned by `groupingBy` is unspecified (backed by a `HashMap`). Wrap it in `new TreeMap<>(result)` if you need sorted keys.</Note>

## counting and other downstream collectors

`Collectors.counting()` is a downstream collector that counts elements in each group. There are several more useful ones:

```java
// Sum the lengths of strings in each group
Map<Character, Integer> totalLength = words.stream()
    .collect(Collectors.groupingBy(
        s -> s.charAt(0),
        Collectors.summingInt(String::length)
    ));

// Join strings in each group with a comma
Map<Character, String> joined = words.stream()
    .collect(Collectors.groupingBy(
        s -> s.charAt(0),
        Collectors.joining(", ")
    ));
```

## partitioningBy

`partitioningBy(predicate)` is like `groupingBy` with exactly two groups — `true` and `false`:

```java
List<Integer> nums = List.of(1, 2, 3, 4, 5, 6, 7, 8);

Map<Boolean, List<Integer>> partition = nums.stream()
    .collect(Collectors.partitioningBy(n -> n % 2 == 0));

System.out.println(partition.get(true));  // [2, 4, 6, 8]
System.out.println(partition.get(false)); // [1, 3, 5, 7]
```

Both keys always exist in the result map (even if one group is empty), so `partition.get(true)` is always safe — no null check needed.

## toMap

`toMap(keyMapper, valueMapper)` builds a `Map<K, V>` from a stream:

```java
Map<String, Integer> wordLengths = Stream.of("apple", "fig", "banana")
    .collect(Collectors.toMap(
        s -> s,           // key: the word itself
        String::length    // value: its length
    ));
// {apple=5, fig=3, banana=6}
```

When duplicate keys are possible, supply a merge function as the third argument:

```java
Map<Integer, String> byLen = Stream.of("hi", "bye", "ok")
    .collect(Collectors.toMap(
        String::length,
        s -> s,
        (existing, incoming) -> existing + "," + incoming // merge duplicates
    ));
```

## Try it

Explore all three major patterns on a word list:

```java
import java.util.*;
import java.util.stream.*;

public class Main {
    public static void main(String[] args) {
        List<String> words = List.of(
            "the", "quick", "brown", "fox", "jumps", "over", "the", "lazy", "dog"
        );

        // Group by first letter
        Map<Character, List<String>> grouped = words.stream()
            .collect(Collectors.groupingBy(s -> s.charAt(0)));
        new TreeMap<>(grouped).forEach((k, v) -> System.out.println(k + ": " + v));

        System.out.println("---");

        // Join with delimiter
        String sentence = words.stream()
            .distinct()
            .sorted()
            .collect(Collectors.joining(" | "));
        System.out.println(sentence);

        System.out.println("---");

        // Partition by length >= 4
        Map<Boolean, List<String>> parts = words.stream()
            .collect(Collectors.partitioningBy(s -> s.length() >= 4));
        System.out.println("long: "  + parts.get(true));
        System.out.println("short: " + parts.get(false));
    }
}
```

## Common mistakes

- **Duplicate keys in `toMap`** — without a merge function, `toMap` throws `IllegalStateException` on any duplicate key. Always supply a merge function when duplicates are possible.
- **Modifying a `toUnmodifiableList()` result** — it throws `UnsupportedOperationException`. Use `Collectors.toList()` if mutability is needed.
- **Expecting `groupingBy` to return a sorted map** — the backing map is a `HashMap`. Wrap in `new TreeMap<>()` for sorted keys.
- **Forgetting that `partitioningBy` always returns both keys** — unlike `groupingBy`, both `true` and `false` keys are always present. You never need to check for null.

## Recap

- `Collectors.toList()` — mutable; `toUnmodifiableList()` / `toList()` — immutable
- `joining(delim, prefix, suffix)` — concatenates strings with optional wrapping
- `groupingBy(fn)` — groups into `Map<K, List<T>>`; downstream collectors further process each group
- `counting()`, `summingInt()`, `mapping()` are common downstream collectors
- `partitioningBy(pred)` — two-key map (`true`/`false`), both keys always present
- `toMap(key, value, merge)` — build arbitrary maps; always provide a merge function for safety
