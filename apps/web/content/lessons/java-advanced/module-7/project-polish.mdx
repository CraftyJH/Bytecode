## Command pattern — undo/redo

Every time a user creates or updates a task, they might want to undo it. The **Command pattern** encapsulates operations as objects:

```java
package com.bytecode.tasks.event;

/** A reversible operation on the task system. */
public interface TaskCommand {
    void execute();
    void undo();
    String description();
}
```

Concrete commands carry the data needed to both apply and reverse the operation:

```java
package com.bytecode.tasks.event;

import com.bytecode.tasks.domain.*;
import com.bytecode.tasks.repository.InMemoryTaskRepository;

import java.util.UUID;

public class CreateTaskCommand implements TaskCommand {

    private final InMemoryTaskRepository repo;
    private final Task task;

    public CreateTaskCommand(InMemoryTaskRepository repo, Task task) {
        this.repo = repo;
        this.task = task;
    }

    @Override public void execute()           { repo.save(task); }
    @Override public void undo()              { repo.delete(task.id()); }
    @Override public String description()     { return "Create task: " + task.title(); }
}

public class UpdateStatusCommand implements TaskCommand {

    private final InMemoryTaskRepository repo;
    private final UUID                   taskId;
    private final TaskStatus             previousStatus;
    private final TaskStatus             newStatus;

    public UpdateStatusCommand(InMemoryTaskRepository repo, UUID taskId,
                               TaskStatus previous, TaskStatus next) {
        this.repo           = repo;
        this.taskId         = taskId;
        this.previousStatus = previous;
        this.newStatus      = next;
    }

    @Override
    public void execute() {
        repo.findById(taskId).ifPresent(t -> repo.save(t.withStatus(newStatus)));
    }

    @Override
    public void undo() {
        // Directly force the status back (bypasses transition guard — undo is privileged)
        repo.findById(taskId).ifPresent(t -> {
            Task reverted = new Task(t.id(), t.title(), t.description(),
                                     previousStatus, t.priority(),
                                     t.assigneeId(), t.dueDate(), t.createdAt());
            repo.save(reverted);
        });
    }

    @Override public String description() {
        return "Status " + previousStatus + " → " + newStatus + " (task " + taskId + ")";
    }
}
```

The `UndoableTaskService` wraps `TaskService` and maintains a history stack:

```java
package com.bytecode.tasks.service;

import com.bytecode.tasks.event.TaskCommand;

import java.util.ArrayDeque;
import java.util.Deque;
import java.util.Optional;

public class UndoableTaskService {

    private final TaskService        delegate;
    private final Deque<TaskCommand> history = new ArrayDeque<>();
    private static final int         MAX_HISTORY = 50;

    public UndoableTaskService(TaskService delegate) {
        this.delegate = delegate;
    }

    private void record(TaskCommand cmd) {
        cmd.execute();
        history.push(cmd);
        if (history.size() > MAX_HISTORY) {
            // Remove oldest (bottom of stack)
            ((ArrayDeque<TaskCommand>) history).removeLast();
        }
    }

    public Optional<Task> createTask(String title, String description,
                                     Priority priority,
                                     Optional<String> assigneeId,
                                     Optional<LocalDate> dueDate) {
        return delegate.createTask(title, description, priority, assigneeId, dueDate)
            .map(task -> {
                history.push(new CreateTaskCommand(taskRepo, task));
                return task;
            });
    }

    public boolean undo() {
        if (history.isEmpty()) return false;
        history.pop().undo();
        return true;
    }

    public String lastAction() {
        return history.isEmpty() ? "(none)" : history.peek().description();
    }
}
```

<Note>Undo operations bypass domain transition guards intentionally — they are a privileged, system-level reversal, not a user-initiated status change. Document this clearly in the Javadoc of `undo()`.</Note>

## Observer pattern — the TaskEventBus

The **Observer pattern** decouples event producers from event consumers. Our generic `TaskEventBus` (which mirrors the EventBus from Module 1) broadcasts domain events to registered listeners:

```java
package com.bytecode.tasks.event;

import java.util.*;
import java.util.function.Consumer;

public class TaskEventBus {

    // Sealed hierarchy of domain events
    public sealed interface TaskEvent permits
        TaskEvent.TaskCreated, TaskEvent.StatusChanged, TaskEvent.TaskAssigned {

        record TaskCreated(Task task)                           implements TaskEvent {}
        record StatusChanged(Task task, TaskStatus previous)   implements TaskEvent {}
        record TaskAssigned(Task task, String assigneeId)      implements TaskEvent {}
    }

    private final Map<Class<?>, List<Consumer<TaskEvent>>> listeners = new HashMap<>();

    public <T extends TaskEvent> void on(Class<T> type, Consumer<T> listener) {
        listeners.computeIfAbsent(type, k -> new ArrayList<>())
                 .add(e -> listener.accept(type.cast(e)));
    }

    public void emit(TaskEvent event) {
        List<Consumer<TaskEvent>> handlers = listeners.get(event.getClass());
        if (handlers != null) {
            handlers.forEach(h -> h.accept(event));
        }
    }
}
```

Wiring it to `TaskService`:

```java
// In TaskService.createTask:

```

Subscribing in the notification layer:

## Builder pattern — composable task queries

The **Builder pattern** constructs complex queries incrementally without a constructor with 10 parameters:

```java
package com.bytecode.tasks.service;

import com.bytecode.tasks.domain.*;

import java.time.LocalDate;
import java.util.List;
import java.util.function.Predicate;

public class TaskQueryBuilder {

    private Predicate<Task> filter = t -> true;  // identity: accept everything

    public TaskQueryBuilder withStatus(TaskStatus status) {
        filter = filter.and(t -> t.status() == status);
        return this;
    }

    public TaskQueryBuilder withPriority(Priority priority) {
        filter = filter.and(t -> t.priority() == priority);
        return this;
    }

    public TaskQueryBuilder assignedTo(String assigneeId) {
        filter = filter.and(t -> t.assigneeId()
                                  .map(a -> a.equals(assigneeId))
                                  .orElse(false));
        return this;
    }

    public TaskQueryBuilder dueBefore(LocalDate date) {
        filter = filter.and(t -> t.dueDate()
                                  .map(d -> d.isBefore(date))
                                  .orElse(false));
        return this;
    }

    public TaskQueryBuilder notDone() {
        filter = filter.and(t -> t.status() != TaskStatus.DONE);
        return this;
    }

    /** Executes the query against the given repository. */
    public List<Task> execute(InMemoryTaskRepository repo) {
        return repo.findWhere(filter);
    }

    /** Returns the composed predicate for reuse outside the builder. */
    public Predicate<Task> build() {
        return filter;
    }
}
```

Usage is expressive and readable:

```java
List<Task> urgent = new TaskQueryBuilder()
    .withStatus(TaskStatus.TODO)
    .withPriority(Priority.CRITICAL)
    .dueBefore(LocalDate.now().plusDays(3))
    .execute(taskRepo);
```

## Javadoc — making the API self-documenting

Production-quality code documents its public API. Key rules:

```java
/**
 * Creates a new task and persists it if all validation rules pass.
 *
 * <p>Validation rules applied (in order):
 * <ol>
 *   <li>Title must not be null or blank.</li>
 *   <li>Due date, if present, must not be in the past.</li>
 *   <li>Priority must not be null.</li>
 * </ol>
 *
 * @param title       the task title; must not be blank
 * @param description a longer description; may be empty
 * @param priority    the task priority; must not be null
 * @param assigneeId  the id of the assigned user, or empty if unassigned
 * @param dueDate     the deadline, or empty if open-ended
 * @return an {@link Optional} containing the persisted task, or
 *         {@code Optional.empty()} if any validation rule fails
 * @see TaskValidation#ALL
 */
public Optional<Task> createTask(String title, String description,
                                 Priority priority,
                                 Optional<String> assigneeId,
                                 Optional<LocalDate> dueDate) { ... }
```

Javadoc rules for this project:
- Every `public` method on a service or repository gets a `@param`, `@return`, and (where relevant) `@throws` tag.
- Domain records can use shorter one-line summaries.
- Internal helpers (`private`) need only a brief comment, not a full doc block.

## Performance review — what to profile in production

When this system moves to production you would instrument these hotspots:

| Hotspot | Why | Fix |
|---|---|---|
| `findAll()` + stream filter | Scans every task on every query | Add an index Map<TaskStatus, Set<UUID>> |
| `findOverdue` in the scheduler | Runs every minute across all tasks | Cache result with a 30-second TTL |
| `notifyAsync` under load | Unbounded `CachedThreadPool` can spawn thousands of threads | Use a bounded pool with back-pressure |
| `serialize Task record` to JSON | Reflection-heavy serialisers are slow | Use a code-generated serialiser (Jackson with `@JsonCreator`) |

Profiling steps: run with `-Xss512k` to surface deep stacks, use `jfr` (Java Flight Recorder) for allocation profiling, and check `jstack` output when the scheduler thread is stuck.

## The complete wired-up system

```java
public class TaskManagementSystem implements AutoCloseable {

    private final InMemoryTaskRepository    taskRepo;
    private final InMemoryProjectRepository projectRepo;
    private final TaskEventBus              eventBus;
    private final TaskService               taskService;
    private final UndoableTaskService       undoable;
    private final NotificationService       notifier;
    private final TaskNotificationService   scheduler;
    private final TaskProcessor             processor;

    public TaskManagementSystem() {
        this.taskRepo    = new InMemoryTaskRepository();
        this.projectRepo = new InMemoryProjectRepository();
        this.eventBus    = new TaskEventBus();
        this.taskService = new TaskService(taskRepo, projectRepo, eventBus);
        this.undoable    = new UndoableTaskService(taskService, taskRepo);
        Executor pool    = Executors.newCachedThreadPool();
        this.notifier    = new NotificationService(pool);
        this.scheduler   = new TaskNotificationService(taskRepo, notifier);
        this.processor   = new TaskProcessor(4, 100);

        // Wire up event-driven notifications
        eventBus.on(TaskEventBus.TaskEvent.TaskCreated.class,
                    e -> processor.enqueue(e.task()));
        eventBus.on(TaskEventBus.TaskEvent.StatusChanged.class,
                    e -> { if (e.task().status() == TaskStatus.DONE)
                               notifier.notifyAsync(e.task()); });

        scheduler.start(60, TimeUnit.SECONDS);
    }

    public UndoableTaskService tasks() { return undoable; }
    public TaskQueryBuilder    query() { return new TaskQueryBuilder(); }

    @Override
    public void close() throws Exception {
        scheduler.close();
        processor.close();
    }
}
```

## Common mistakes

- **Commands that capture mutable state** — a `CreateTaskCommand` that stores a reference to a mutable `Task` (not a record) risks the undo using stale data if the task is modified after the command is created. Always capture an immutable snapshot in the command's constructor.
- **Event bus without error isolation** — if one subscriber throws an exception, a naive event bus propagates it to the publisher and skips remaining subscribers. Wrap each `h.accept(event)` call in a try-catch that logs the exception and continues to the next handler.
- **Builder that mutates the same Predicate** — `filter = filter.and(...)` creates a *new* composed predicate each time; it does not mutate the original. This is correct. If you mistakenly store predicates as mutable fields and try to reset them by assigning `filter = null`, the builder becomes non-reusable.
- **Javadoc that repeats the code** — `@param title the title` documents nothing. Write what constraints apply (`must not be blank`), what the absence means for the return value, and cross-reference related methods with `@see`.

## Recap

- The **Command pattern** captures operations as first-class objects with `execute`/`undo`, enabling an arbitrary-depth undo stack without modifying the original service.
- The **Observer pattern** via a typed `TaskEventBus` decouples event producers (services) from consumers (notifications, processors) — adding a new subscriber requires zero changes to the service.
- The **Builder pattern** with composed `Predicate<T>` creates readable, chainable query objects that replace multi-parameter methods.
- **Javadoc** on public APIs communicates contracts — preconditions, return semantics, and cross-references — that code alone cannot express.
- Performance profiling targets: unbounded collections, hot O(n) scans, and unbound thread pools are the three most common production pain points in systems like this one.
