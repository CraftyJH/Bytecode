In the previous lesson you created a Dog and then assigned its fields one line at a time. That's tedious, easy to forget, and leaves the object in a half-built state between assignments. **Constructors** solve this: they're special methods that run automatically when an object is created, so you can supply all the data up front.

Think of a constructor like a factory's assembly line. When you order `new Dog("Rex", 4)`, you're placing an order with the factory. The factory (constructor) takes your specifications, runs through the setup steps, and hands you back a fully built Dog — not a box of unassembled parts.

## A constructor is a method named after the class

A constructor has the same name as the class, no return type, and runs once per object — when `new` is used:

```java
class Dog {
    String name;
    int age;

    Dog(String name, int age) {
        this.name = name;
        this.age = age;
    }
}
```

Now creation is one line:

```java
Dog rex = new Dog("Rex", 4);
System.out.println(rex.name + " is " + rex.age);  // Rex is 4
```

The arguments to `new Dog(...)` are passed to the constructor's parameters. The constructor body assigns them to fields, and the freshly built object is handed back.

## What `this` is for

When the parameter and the field share a name, plain `name` refers to the parameter (the closer scope wins). To reach the field, you need an explicit reference to the current object:

```java
Dog(String name, int age) {
    this.name = name;   // this.name is the field, name is the parameter
    this.age = age;
}
```

`this` is an implicit reference to the object whose constructor (or instance method) is running. You can also use it explicitly to call other methods or read other fields on the same object.

<Note>You can avoid `this` entirely by naming the parameters differently — `Dog(String n, int a)` and then `name = n;`. Most Java code prefers the matching-name + `this.` style; it makes the field assignments obvious at a glance.</Note>

## The default constructor

If you write a class with **no** constructors, Java provides a hidden no-argument constructor for free. That's why `new Dog()` worked in the previous lesson — Java synthesized `Dog() { }` for you. The moment you define any constructor of your own, that free one disappears:

```java
class Dog {
    String name;

    Dog(String name) {       // we defined a one-arg constructor
        this.name = name;
    }
}

// new Dog();           // compile error — no zero-arg constructor exists
new Dog("Rex");          // OK
```

If you want both forms, you can write multiple constructors (constructor overloading works just like method overloading):

```java
class Dog {
    String name;
    int age;

    Dog() {
        this.name = "Unknown";
        this.age = 0;
    }
    Dog(String name, int age) {
        this.name = name;
        this.age = age;
    }
}
```

`new Dog()` calls the first; `new Dog("Rex", 4)` calls the second.

## Constructor chaining with this(...)

To avoid duplicating initialization logic across multiple constructors, one constructor can call another using `this(...)`:

```java
class Dog {
    String name;
    int age;
    String breed;

    Dog(String name, int age) {
        this(name, age, "Unknown");   // delegate to the three-arg version
    }

    Dog(String name, int age, String breed) {
        this.name = name;
        this.age = age;
        this.breed = breed;
    }
}
```

`this(...)` must be the first statement in the constructor body. This pattern keeps all the real initialization in one place and treats the shorter constructors as convenient shorthand.

## Try it

```java
public class Demo {
    public static void main(String[] args) {
        Point p = new Point(10, 20);
        System.out.println(p.x);   // 10
        System.out.println(p.y);   // 20
    }
}

class Point {
    int x, y;
    Point(int x, int y) {
        this.x = x;
        this.y = y;
    }
}
```

## Common mistakes

- **Giving the constructor a return type** → `void Dog(...)` is not a constructor; it's a regular method that happens to share the class's name. Constructors have no return type at all — not even `void`.
- **Assigning a parameter to itself** → without `this.`, `name = name;` is a no-op. The field never gets written. The compiler won't warn you — add `@SuppressWarnings("SelfAssignment")` if you want a lint check.
- **Forgetting that defining one constructor removes the default** → if you write `Dog(String name)`, then `new Dog()` no longer compiles unless you also write a no-arg constructor.
- **Calling another constructor with plain `Dog(...)`** → you can chain to another constructor, but the syntax is `this(...)` on the first line, not `Dog(...)`.

## Recap

Constructors make objects self-consistent from the moment they're created. A well-designed class ensures there's no way to construct an object in an invalid state — the constructor validates inputs or sets safe defaults.

- A constructor has the same name as the class and no return type
- It runs once when `new` is used, and is the right place to initialise fields
- `this.field = parameter;` is the idiom for writing the field when names collide
- Java provides a default no-arg constructor only if you define no constructors yourself
- Multiple constructors with different parameter lists let callers create objects different ways
