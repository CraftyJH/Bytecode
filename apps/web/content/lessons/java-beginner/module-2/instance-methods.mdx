A class without methods is just a record — somewhere to store data. The point of OOP is that objects also **do things**. An **instance method** is a method that runs in the context of one specific object and has free access to that object's fields through `this`.

Think of a bank account object. The account holds a balance (data), but the balance alone isn't very useful. The account also needs to *do things* — deposit, withdraw, check the balance. Those actions are instance methods. They operate on the account they're called on, not on some generic account.

## Defining and calling instance methods

Instance methods look like the static methods you already know, but without the `static` keyword:

```java
class Dog {
    String name;
    int age;

    Dog(String name, int age) {
        this.name = name;
        this.age = age;
    }

    void bark() {
        System.out.println(name + " says woof!");
    }

    void birthday() {
        age++;
    }
}
```

Both `bark` and `birthday` operate on the Dog they were called on. You call them with the dot operator:

```java
Dog rex = new Dog("Rex", 4);
rex.bark();        // Rex says woof!
rex.birthday();
System.out.println(rex.age);  // 5
```

The variable `rex` is implicitly passed in as `this`, so inside the method, `name` is `rex.name` and `age++` increments `rex.age`. If you had a second Dog `buddy`, `buddy.bark()` would print Buddy's name — same code, different object.

<Note>Inside an instance method you can write `name` or `this.name` — both refer to the same field. Use `this.` only when needed to disambiguate from a parameter, or to make the access deliberately explicit.</Note>

## Instance methods that return values

Methods aren't only for mutating state. They can also read it and return a result:

```java
class Rectangle {
    int width, height;
    Rectangle(int w, int h) { this.width = w; this.height = h; }

    int area() {
        return width * height;
    }

    boolean isSquare() {
        return width == height;
    }
}

Rectangle r = new Rectangle(4, 7);
System.out.println(r.area());      // 28
System.out.println(r.isSquare());  // false
```

This pattern — a tiny method that derives one value from the object's fields — is everywhere in Java. It's how you avoid scattering `width * height` across the rest of your code.

## Methods that call other methods on the same object

An instance method can call other instance methods on the same object simply by name:

```java
class Temperature {
    double celsius;

    Temperature(double c) { this.celsius = c; }

    double toFahrenheit() {
        return celsius * 9.0 / 5.0 + 32;
    }

    void printBoth() {
        System.out.println(celsius + "°C = " + toFahrenheit() + "°F");
    }
}
```

Inside `printBoth`, the call `toFahrenheit()` is shorthand for `this.toFahrenheit()` — it calls the method on the same object.

## Static vs instance — when to use which

Instance methods need an object. Static methods don't:

```java
class Math2 {
    static int square(int n) {     // no object needed
        return n * n;
    }
}

Math2.square(5);                   // 25 — call on the class
```

Use **static** for utilities whose result depends only on the inputs (`Math.max`, `Math.sqrt`). Use **instance** when the behavior depends on the object's state (`dog.bark()`, `account.deposit(50)`).

A static method has no `this` — if you try to read an instance field from inside one, you get a compile error.

## Common mistakes

- **Calling an instance method without an object** → `bark();` from outside the class is a compile error. You must write `dog.bark();` so Java knows which Dog's `name` to use.
- **Adding `static` to an instance method by accident** → static methods can't read instance fields. You'll get "non-static field cannot be referenced from a static context". This is the most common error for beginners adding methods to their classes.
- **Returning a value from a `void` method** → declared return type and `return` statement must agree. Either change the type or remove the value.
- **Mutating fields from outside the class when an instance method would be cleaner** → if `birthday()` exists, prefer `dog.birthday()` over `dog.age = dog.age + 1;` from elsewhere. The method centralizes the logic and keeps the change readable.

## Recap

Instance methods are where OOP starts to pay off: instead of scattered functions operating on bare data, you have objects that know how to manage themselves. The `dog.bark()` call reads like English — a specific dog is doing a specific thing.

- Instance methods are defined without `static` and called as `object.method(args)`
- Inside an instance method, fields are read and written through the implicit `this` reference
- Static methods belong to the class; instance methods belong to a specific object
- Methods can return computed values that depend on object state — keep these methods short and focused
