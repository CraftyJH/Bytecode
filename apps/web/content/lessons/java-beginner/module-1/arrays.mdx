An array is a fixed-size, ordered container that holds multiple values of the same type. Instead of declaring ten separate variables for ten scores, you declare one array. That makes your code shorter, and more importantly, it lets you process all the values with a loop.

Think of an array like a row of numbered mailboxes in an apartment building. Each box has an address (the index), holds exactly one item, and the total number of boxes is fixed when the building is constructed. You can look up any box instantly by its number.

## Declaring and initializing arrays

There are two common ways to create an array:

```java
// Allocate an array of 5 ints (all start at 0)
int[] scores = new int[5];
scores[0] = 88;
scores[1] = 92;
scores[2] = 75;
scores[3] = 100;
scores[4] = 83;

// Array literal — when you know the values upfront
String[] days = {"Mon", "Tue", "Wed", "Thu", "Fri"};
```

Arrays are **zero-indexed**: the first element is at index 0, and the last is at index `array.length - 1`. Accessing an index outside this range throws an `ArrayIndexOutOfBoundsException` at runtime.

## Iterating over arrays

Use a standard `for` loop when you need the index, and an enhanced `for-each` loop when you only need the values:

```java
int[] temps = {72, 68, 75, 80, 65};

// With index
for (int i = 0; i < temps.length; i++) {
    System.out.println("Day " + i + ": " + temps[i]);
}

// For-each — cleaner when index isn't needed
for (int temp : temps) {
    System.out.println(temp);
}
```

The for-each loop reads: "for each `int` called `temp` in the array `temps`". You cannot use it to modify the array elements or when you need the current index.

## Array length

`array.length` is a field (not a method — no parentheses). It is fixed at the moment the array is created and cannot change.

```java
int[] data = new int[10];
System.out.println(data.length); // 10
```

If you need a resizable collection, you'll want `ArrayList` — that's covered in a later module.

<Note>`array.length` (no parentheses) is a field on arrays. `string.length()` (with parentheses) is a method on Strings. This inconsistency trips up almost every Java beginner — you're not alone if you mix them up.</Note>

## Modifying array elements

Array elements are like regular variables: you can reassign them at any time by index.

```java
int[] values = {1, 2, 3, 4, 5};
values[2] = 99;   // replace the third element

for (int v : values) {
    System.out.print(v + " ");
}
// Output: 1 2 99 4 5
```

Remember that the for-each loop variable is a *copy* — assigning to it does not change the array. You need an index-based loop if you want to modify elements.

## Try it

```java
public class ArrayAverage {
    public static void main(String[] args) {
        int[] grades = {85, 92, 78, 90, 88};
        int sum = 0;

        for (int grade : grades) {
            sum += grade;
        }

        double average = (double) sum / grades.length;
        System.out.println("Average: " + average); // Average: 86.6
    }
}
```

## Common mistakes

- **ArrayIndexOutOfBoundsException** → using `<= array.length` instead of `< array.length` in a loop is the most common cause. Valid indices are always `0` through `length - 1`, not `length` itself.
- **Confusing `.length` with `.length()`** → arrays use `.length` (field, no parentheses); Strings use `.length()` (method, with parentheses). The JVM will tell you clearly when you've mixed them up.
- **Modifying elements in a for-each** → the loop variable is a copy of the element. `for (int x : arr) { x = 0; }` does nothing to the array. Use `arr[i] = 0` with an index-based loop instead.
- **Assuming arrays are initialized to your value** → numeric arrays default to `0`, boolean arrays to `false`, and object arrays to `null`. Verify before reading if you need non-default starting values.

## Recap

Arrays are the simplest way to manage a collection of values in Java. Their fixed size and zero-indexing take a little getting used to, but once they click you'll use them constantly — they underlie most data structures and algorithms you'll encounter.

- Declare with `Type[] name = new Type[size]` or an array literal `{v1, v2, v3}`
- Arrays are zero-indexed; valid indices are `0` to `length - 1`
- `array.length` (no parentheses) gives the fixed number of elements
- Use a standard `for` loop when you need the index; use for-each when you only need values
- Accessing an out-of-bounds index throws `ArrayIndexOutOfBoundsException` at runtime
