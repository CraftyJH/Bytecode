Java is the language behind a huge portion of the world's software — bank backends, Android apps, the systems that route your flights and process your card payments. It was designed in 1995 around a simple promise: *write once, run anywhere.* That promise is why Java still powers so much critical infrastructure thirty years later.

## Every Java program starts with a class

Java organizes all code into **classes**. A class is just a named container for code. Inside the class, the special method `main` is where the program starts running.

```java
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, Java");
    }
}
```

Three things to notice:

- `public class HelloWorld` — the container, named whatever we want
- `public static void main(String[] args)` — the special starting point. Memorize this signature; we'll explain every word over the next few lessons.
- `System.out.println(...)` — prints text to the screen, then moves to a new line

## Common mistakes

- **Forgetting the semicolon** → `error: ';' expected`. Every statement ends with `;`.
- **Single quotes around text** → in Java, single quotes are for single characters (`'A'`), not strings. Use `"double quotes"` for text.
- **Misspelling `main`** → the program compiles but won't run as a standalone application.

## Recap

- Java code lives in **classes**
- `public static void main(String[] args)` is where execution starts
- `System.out.println(...)` prints a line of text
- Statements end with `;`
