Java is a **statically typed** language — before you can store a value, you must tell the compiler exactly what kind of value it is. This might feel strict coming from other languages, but it catches entire categories of bugs before your program ever runs.

## Declaring variables

The syntax is always: `type name = value;`

```java
int age = 25;
double price = 9.99;
boolean isLoggedIn = true;
String username = "alice";
```

Java has exactly **eight primitive types** built into the language. In practice you'll use four of them almost exclusively:

| Type | Use for | Example |
|------|---------|---------|
| `int` | whole numbers | `int count = 0;` |
| `double` | decimal numbers | `double pi = 3.14;` |
| `boolean` | true / false flags | `boolean done = false;` |
| `char` | a single character | `char grade = 'A';` |

The other four — `byte`, `short`, `long`, `float` — exist for specific performance or range needs. `long` needs an `L` suffix on its literal: `long bigNum = 9_000_000_000L;`

`String` is not a primitive (it's a class), but it's so fundamental that it feels like one.

## Constants with final

When a value should never change after it's first assigned, mark it `final`. By convention, constant names are written in SCREAMING_SNAKE_CASE.

```java
final int MAX_RETRIES = 3;
final double TAX_RATE = 0.08;

// This would cause a compile error:
// MAX_RETRIES = 5;
```

## Try it

Declare three variables — one `int`, one `double`, and one `boolean` — then print all three on separate lines.

```java
public class Variables {
    public static void main(String[] args) {
        int score = 42;
        double temperature = 98.6;
        boolean isRaining = false;
        System.out.println(score);
        System.out.println(temperature);
        System.out.println(isRaining);
    }
}
```

## Common mistakes

- **Assigning a decimal to an int** → `int x = 1.5;` is a compile error. Use `double` for decimals.
- **Forgetting `L` on long literals** → `long n = 10000000000;` won't compile. Write `10000000000L`.
- **Using `=` instead of `==`** → `=` assigns a value; `==` compares two values. Mixing them up causes logic bugs or compile errors.
- **Comparing with `==` for non-primitive types** → for `String` and other objects, use `.equals()`, not `==`.

## Recap

- Every variable has a fixed type declared before its name
- The four workhorse primitives: `int`, `double`, `boolean`, `char`
- `String` is a class, not a primitive, but is used like one
- Use `final` to declare constants; name them in SCREAMING_SNAKE_CASE
- `long` literals need an `L` suffix; `float` literals need an `F` suffix
