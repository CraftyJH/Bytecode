Expressions are the calculations at the heart of every Java program. An expression combines values, variables, and operators to produce a new value. Understanding how Java evaluates them — and the surprises that come from integer division — will save you hours of debugging.

## Arithmetic operators

The five arithmetic operators work exactly as you'd expect, with one important exception:

```java
int a = 10;
int b = 3;

System.out.println(a + b);  // 13
System.out.println(a - b);  // 7
System.out.println(a * b);  // 30
System.out.println(a / b);  // 3  ← truncates, not 3.333!
System.out.println(a % b);  // 1  ← remainder
```

**Integer division truncates toward zero.** `7 / 2` gives `3`, not `3.5`. If you need the decimal, make at least one operand a `double`: `7.0 / 2` or `(double) 7 / 2`.

Compound assignment operators are shorthand: `x += 5` is the same as `x = x + 5`. The same pattern works for `-=`, `*=`, `/=`, and `%=`.

## Comparison and logical operators

Comparison operators return a `boolean`:

```java
int score = 85;
System.out.println(score >= 90);   // false
System.out.println(score != 100);  // true
System.out.println(score == 85);   // true
```

Logical operators combine boolean expressions:

```java
boolean isAdult = age >= 18;
boolean hasTicket = true;

System.out.println(isAdult && hasTicket);  // true only if both
System.out.println(isAdult || hasTicket);  // true if either
System.out.println(!isAdult);              // flips the value
```

## String concatenation

The `+` operator doubles as string concatenation. When one side is a `String`, Java converts the other side to a string automatically:

```java
String name = "Alice";
int score = 95;
System.out.println("Player " + name + " scored " + score + " points.");
// Player Alice scored 95 points.
```

Watch out: `"Result: " + 1 + 2` prints `"Result: 12"` (left-to-right string concat), not `"Result: 3"`. Use parentheses to force arithmetic first: `"Result: " + (1 + 2)`.

## Operator precedence

Java follows standard math precedence: `*`, `/`, `%` before `+`, `-`. When in doubt, use parentheses — they make intent clear and eliminate ambiguity.

```java
int result = 2 + 3 * 4;    // 14, not 20
int clear  = (2 + 3) * 4;  // 20
```

## Common mistakes

- **int / int truncates** → `5 / 2` is `2`, not `2.5`. Cast one operand to `double` if you need decimals.
- **String + int + int** → `"x=" + 1 + 2` gives `"x=12"`. Wrap the math in parentheses: `"x=" + (1 + 2)`.
- **Using `=` in a condition** → `if (x = 5)` is a compile error in Java (unlike C). You want `==`.
- **Confusing `&&` and `&`** → `&&` short-circuits (stops early if left side is false); `&` always evaluates both sides.

## Recap

- `+`, `-`, `*`, `/`, `%` are the arithmetic operators; `%` gives the remainder
- `int / int` performs integer division and drops the decimal
- `==`, `!=`, `>`, `<`, `>=`, `<=` compare values and return `boolean`
- `&&` (AND), `||` (OR), `!` (NOT) combine boolean expressions
- `+` concatenates strings; use parentheses to control evaluation order
