Programs that only run top-to-bottom aren't very useful. Control flow lets you make decisions, choose between paths, and handle cases — the fundamental skill that separates a working program from a useful one.

## if / else if / else

The `if` statement evaluates a boolean expression and executes a block only when it's true.

```java
int temperature = 22;

if (temperature >= 30) {
    System.out.println("Hot day");
} else if (temperature >= 20) {
    System.out.println("Warm day");
} else if (temperature >= 10) {
    System.out.println("Cool day");
} else {
    System.out.println("Cold day");
}
```

**Always use curly braces `{}`**, even for single-line bodies. Omitting them is legal but causes notorious bugs when you add a second line later and forget to add braces.

Java checks each condition top-to-bottom and stops at the first one that's true. Order matters: put the most specific (or highest-value) conditions first.

Think of an `else-if` chain like a series of filters on a conveyor belt. As soon as a filter catches the value, the rest are skipped. Only one branch runs per execution.

## The ternary operator

The ternary operator is a compact way to pick between two values based on a condition:

```java
// condition ? valueIfTrue : valueIfFalse
int age = 20;
String label = (age >= 18) ? "adult" : "minor";
System.out.println(label);  // adult
```

Use ternary for simple value selection. For anything with side effects or multiple statements, stick with a regular `if/else`.

<Note>Nested ternary expressions — `a ? b : c ? d : e` — are technically valid but quickly become hard to read. If you need more than one level of nesting, switch to an `if/else if/else` chain instead.</Note>

## switch

`switch` compares one variable against multiple constant values. Traditional switch uses `break` to prevent fall-through:

```java
int day = 3;
switch (day) {
    case 1:
        System.out.println("Monday");
        break;
    case 2:
        System.out.println("Tuesday");
        break;
    case 3:
        System.out.println("Wednesday");
        break;
    default:
        System.out.println("Other day");
}
```

Java 14+ introduced **switch expressions** with `->` that are cleaner and don't fall through:

```java
String dayName = switch (day) {
    case 1 -> "Monday";
    case 2 -> "Tuesday";
    case 3 -> "Wednesday";
    default -> "Other day";
};
System.out.println(dayName);
```

Prefer the arrow form when your Java version supports it — it's less error-prone.

## Try it

```java
public class ControlFlow {
    public static void main(String[] args) {
        int score = 85;

        String grade = switch (score / 10) {
            case 10, 9 -> "A";
            case 8     -> "B";
            case 7     -> "C";
            case 6     -> "D";
            default    -> "F";
        };

        System.out.println("Grade: " + grade);
    }
}
```

## Common mistakes

- **Omitting `{}`** → adding a second line to an if body without braces silently breaks the logic. Only the first line is guarded by the condition; the second runs unconditionally. This is one of the most dangerous "looks fine at a glance" bugs.
- **Missing `break` in traditional switch** → execution falls through to the next case automatically. Use the arrow form (`->`) to eliminate this risk entirely, or add `break` to every case.
- **Condition order in else-if** → if a broad condition comes first (`score >= 60`), narrower ones below it (`score >= 90`) are never reached for any score above 60.
- **Assignment in condition** → `if (x = 5)` won't compile in Java, but double-check you mean `==` for comparison.

## Recap

Branching logic is how programs adapt to their inputs. The single most important habit here is always using braces — the short-term convenience of dropping them is not worth the long-term debugging pain.

- `if / else if / else` selects a branch based on boolean conditions — evaluated top to bottom
- Always wrap bodies in `{}`, even single statements
- The ternary `condition ? a : b` is a concise value selector, not a replacement for complex logic
- Traditional `switch` requires `break`; the modern arrow form (`->`) does not fall through
- Order conditions from most specific to least specific
