Reading a text file is one of those operations that should be a one-liner — and with NIO.2 it finally is. This lesson covers the three calls you'll reach for ninety percent of the time.

## The two main reads

```java
import java.nio.file.*;
import java.util.*;

Path p = Paths.get("/tmp/poem.txt");

// 1) Read everything into one String (Java 11+)
String all = Files.readString(p);

// 2) Read line-by-line into a List
List<String> lines = Files.readAllLines(p);
```

Both methods open the file, read it fully, and close it before returning — you do not need to manage resources or worry about closing anything. The catch: both load the entire file into memory.

`readString` returns the file as one String, including any line terminators. `readAllLines` splits on Unix (`\n`), Windows (`\r\n`), or Mac (`\r`) line endings — the result list contains lines *without* the terminators.

<Note>Both methods throw <code>IOException</code> (a checked exception) if the file doesn't exist, can't be read, or contains invalid bytes for the chosen charset. Either declare <code>throws IOException</code> on your method or wrap the call in try-catch.</Note>

## Encoding matters

`Files.readString` and `Files.readAllLines` default to **UTF-8**. That's almost always what you want — UTF-8 is the dominant text encoding on the modern web. But if you're reading a file that was written in a different encoding, pass a `Charset`:

```java
import java.nio.charset.*;

String latin = Files.readString(p, StandardCharsets.ISO_8859_1);
```

Mismatching the charset doesn't crash — it silently produces garbled text (mojibake). When you see strange characters appearing in your strings, encoding is the first thing to suspect.

## Reading binary data

For non-text files (images, PDFs, compiled bytecode) use `Files.readAllBytes`:

```java
byte[] bytes = Files.readAllBytes(Paths.get("/tmp/photo.jpg"));
System.out.println("Loaded " + bytes.length + " bytes");
```

There's no encoding step — you get the raw bytes from disk.

## When NOT to use these methods

`readString`, `readAllLines`, and `readAllBytes` all fully materialise the file in memory. That's perfect for small files (configs, small CSVs, short logs) but problematic for anything that might exceed your heap.

For files that are large or might grow unboundedly, prefer either:

- **`Files.lines(path)`** — returns a `Stream<String>` that reads lazily, line by line. Covered in a later lesson.
- **`Files.newBufferedReader(path)`** — a `BufferedReader` you drive with `readLine()` in a loop. Covered alongside try-with-resources.

A handy rule of thumb: if the file might exceed roughly 100 MB, don't `readString` it. Stream it instead.

## Buffering is automatic

You may have noticed there's no `BufferedReader` in any of these examples. That's deliberate — `Files.readString` and `Files.readAllLines` already read in large chunks under the hood. You only need to think about buffering when you go to the lower-level `BufferedReader`/`InputStreamReader` API directly.

## Common mistakes

- **Forgetting to declare `throws IOException`** — every read method is checked. The compiler will refuse to let you ignore the possibility of failure.
- **Using `readString` on huge files** — easy to do, hard to debug when your program suddenly OOMs in production. If a file's size is bounded by user input, assume it can be large.
- **Trusting the default encoding** — Java's older APIs (like `new FileReader(file)`) used the platform default charset, which led to "works on my machine" bugs. NIO.2 string methods use UTF-8 by default; older code may not.
- **Not handling missing files** — `NoSuchFileException` (a subclass of IOException) is thrown when the file doesn't exist. Catch it explicitly if your program needs to give a friendlier error.

## Recap

Reading a text file in modern Java should be one line of code. The NIO.2 API lives up to that promise — once you know the three calls, almost every "load this file" task reduces to a one-liner.

- **`Files.readString(path)`** — entire file as one String (Java 11+).
- **`Files.readAllLines(path)`** — file as a `List<String>` of terminator-free lines.
- **`Files.readAllBytes(path)`** — entire file as a `byte[]`, for binary content.
- All three default to UTF-8 (where applicable). Pass a `Charset` to override.
- All three load the whole file into memory; for large files use `Files.lines` or a `BufferedReader`.
